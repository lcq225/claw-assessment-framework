# ClawAF - Claw评价框架

> 用于评价基于Claw体系的AI Agent系统智慧能力的标准化框架

---

## 概述

**ClawAF (Claw Assessment Framework)** 是一套标准化评价框架，用于评估基于Claw体系的AI Agent系统的智慧能力，包括 OpenClaw、CoPaw、qclaw、winClaw、zeroclaw 等所有变种。

### 目标

提供通用的AI Agent能力测量标准，支持：
- 自我评估
- 团队基准测试
- 进化路线规划
- 行业水平对比

---

## 快速开始

### 安装

```bash
git clone https://github.com/lcq225/claw-assessment-framework.git
cd claw-assessment-framework
pip install -r requirements.txt
```

### 自动评估（推荐）

脚本会自动扫描您的Claw工作区目录，基于实际文件和配置进行评价：

```bash
python assess_auto.py
```

或指定您的Claw工作区目录：

```bash
# CoPaw 示例
python assess_auto.py --dir "D:\CoPaw\.copaw\workspaces\default"

# OpenClaw 示例
python assess_auto.py --dir "/path/to/openclaw/.copaw/workspaces/default"

# Linux/Mac 示例
python assess_auto.py --dir "~/copaw/.copaw/workspaces/default"
```

**重要说明：** `--dir` 参数应指向**工作区目录**（包含 `AGENTS.md`、`SOUL.md`、`PROFILE.md`、`skills/` 的文件夹），而不是项目根目录。

**Claw 目录结构：**

```
项目根目录/
├── .copaw/
│   ├── workspaces/
│   │   ├── default/           <-- 评估这个目录（工作区目录）
│   │   │   ├── AGENTS.md
│   │   │   ├── SOUL.md
│   │   │   ├── PROFILE.md
│   │   │   └── skills/
│   │   └── other-workspace/
│   └── skills/               <-- 全局技能（可选）
└── ... (项目文件)
```

### 目录诊断工具

不确定哪个目录应该评估？使用诊断工具自动查找：

```bash
python diagnose.py --dir "D:\CoPaw"
```

诊断工具会：
1. 检查当前目录是否为工作区
2. 搜索所有可用的工作区目录
3. 提供正确的评估命令

示例输出：
```
Found workspace directories:

1. D:\CoPaw\.copaw\workspaces\default
   Command: python assess_auto.py --dir "D:\CoPaw\.copaw\workspaces\default"
   Files: AGENTS.md SOUL.md PROFILE.md

Recommended: Use the first workspace directory
```

**工作原理：**

1. **自动检测** - 自动查找您的Claw目录
2. **文件扫描** - 扫描关键文件（AGENTS.md、memory.db、skills等）
3. **数据库查询** - 查询记忆数据库获取记录数量
4. **客观评分** - 基于实际数据评分，非手工输入
5. **详细结果** - 显示各维度的检测点
6. **双语支持** - 支持中英文双语关键词检测，适用于国际化团队

输出：

### 转换结果为其他格式

生成JSON结果后，转换为人类可读的格式：

```bash
# 英文版本
python format_result.py clawaf_result.json

# 中文版本
python format_result_cn.py clawaf_result.json
```

这将生成：
- `clawaf_result.json` - 机器可读（程序处理、对比、历史记录）
- `clawaf_result.md` - 人类可读（查看、分享、归档）
- `clawaf_result.html` - 可视化报告（浏览器查看、演示）
- `clawaf_result_cn.*` - 中文版本

**格式对比：**

| 格式 | 最适合 | 使用场景 |
|------|--------|----------|
| **JSON** | 机器 | 程序处理、对比、历史追踪 |
| **Markdown** | 人类 | 快速阅读、文档归档、版本控制 |
| **HTML** | 可视化 | 演示、分享、打印 |

## 8大评价维度

### 1. 身份认知度

AI Agent 是否建立了清晰、完整的自我认知？

| 评分 | 描述 |
|------|------|
| 0-20 | 无身份认知，仅基础模板 |
| 21-40 | 有基础身份信息（名字、角色） |
| 41-60 | 有完整身份文档（AGENTS.md） |
| 61-80 | 身份文档详细，包含规则、偏好 |
| 81-100 | 身份认知完整，持续进化（AGENTS.md/SOUL.md/PROFILE.md） |

**评估指标：**
- 身份文档完整性
- 规则清晰度
- 偏好明确度
- 持续更新频率

## 背景与目的

### 背景

AI Agent 系统正在快速发展，但缺乏标准化的能力评估方法。不同的实现（OpenClaw、CoPaw 等）各有优劣，导致：

1. 无法客观对比不同系统
2. 难以测量进步
3. 无法识别改进方向
4. 难以跨团队分享最佳实践

### 目的

ClawAF 旨在：

1. **标准化评价** - 提供通用的AI Agent能力评估框架
2. **支持对比** - 实现不同系统和团队间的客观对比
3. **引导进化** - 识别改进领域，提供可操作的建议
4. **分享知识** - 建立社区，分享最佳实践和经验

### 原则

1. **实用性优先** - 标准来自实际协作经验
2. **可量化** - 所有维度都有清晰的量化指标
3. **可扩展** - 框架可适应不同场景
4. **社区驱动** - 开源，鼓励社区贡献

## 结果应用

### 1. 自我改进

根据评估结果：
- **低于平均分的维度** → 优先改进
- **接近下一等级的维度** → 持续优化
- **已满分的维度** → 分享经验

### 2. 团队基准测试

- 识别团队优缺点
- 建立内部基准
- 跨成员分享最佳实践

### 3. 进化规划

## 核心脚本说明

### 评估脚本

- **assess_auto.py** - 自动评估脚本（推荐）
  - 自动扫描Claw工作区目录
  - 检测文件、目录和数据库记录
  - 基于实际数据客观评分
  - 支持中英文双语关键词检测
  - 无需手工输入

- **assess.py** - 手动自我评估脚本
  - 交互式问卷
  - 适合文件结构不完整的团队

- **compare.py** - 对比脚本
  - 对比两个评估结果
  - 显示分数变化和维度差异
  - 详细改进洞察

### 格式化脚本

- **format_result.py** - 结果格式化（英文）
  - 将JSON转换为Markdown和HTML
  - 生成人类可读报告
  - 支持进度条和可视化元素

- **format_result_cn.py** - 结果格式化（中文）
  - 将JSON转换为中文Markdown和HTML
  - 翻译维度名称和等级
  - 支持中文用户

- **README.md** - 英文文档
